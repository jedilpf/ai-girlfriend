## 一、产品定位

**一句话：** 一个能和你聊天、有性格、会记住你的 AI 女友。

**核心卖点：**
- MBTI 16 种人格可选，每种性格完全不同
- 能语音说话，不只是打字
- 有好感度，越聊越亲近
- 可以夸她、奖励她，她会有反应
- 她会主动找你说话（Demo 阶段模拟）

**不做的事（Demo 阶段）：**
- ❌ 原生 APP（以后套壳就行）
- ❌ 本地跑模型（API 更快）
- ❌ 多人/社交（单人体验）
- ❌ 真正的推送通知（需要后端，Demo 阶段模拟）
- ❌ 付费/商业化

## 三、架构安全说明

### 🔴 为什么需要代理层

纯前端直接调 AI API 有两个致命问题：

1. **API Key 暴露** — 任何人 F12 就能看到你的 Key，盗用额度
2. **CORS 拦截** — 通义 DashScope 和 ElevenLabs 的 API **不允许浏览器直接调用**

### 解决方案：Vercel Edge Function

```
浏览器 (PWA)
    ↓ fetch('/api/chat')
Vercel Edge Function（注入 API Key，转发请求）
    ↓
通义 DashScope API
    ↓ 返回结果
Vercel Edge Function
    ↓ 返回给前端
浏览器
```

- API Key 存在 Vercel 环境变量中，前端永远接触不到
- CORS 由 Vercel 处理，浏览器只和自己的域名通信
- Vercel Hobby 计划免费，足够 Demo 使用

### 前端调用方式

```typescript
// ❌ 错误：直接调第三方 API，Key 暴露 + CORS 报错
const response = await fetch('https://dashscope.aliyuncs.com/...', {
  headers: { Authorization: `Bearer ${API_KEY}` }
});

// ✅ 正确：调自己的 Edge Function
const response = await fetch('/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ messages, model: 'qwen-plus' }),
});
```

## 四、功能清单（按优先级）

### P0 - 必须有的（今天做完就能用）

#### 1. 角色选择页
- 展示 4 个预设角色（先做 4 个，不全 16 个）
- 每个角色显示：头像（占位）、名字、MBTI 类型、一句话介绍
- 点击选择后进入聊天

**建议先做的 4 个：**

| 名字 | MBTI | 性格标签 | 一句话 |
|------|------|---------|--------|
| 小暖 | ENFJ | 温柔学姐 | "今天过得怎么样呀？" |
| 傲娇 | INTP | 毒舌理工女 | "才不是特意等你的。" |
| 元气 | ESFP | 快乐小狗 | "啊啊啊你来啦！！" |
| 御姐 | INTJ | 高冷总裁 | "说吧，什么事。" |

#### 2. 聊天界面
- 消息气泡（左=她，右=你）
- 输入框 + 发送按钮
- 语音输入按钮（🎤，不支持时自动隐藏）
- 顶部显示角色名 + 好感度
- 打字效果（逐字显示）
- **Loading 状态**：AI 回复中显示"正在输入..."动画
- **错误状态**：API 超时/失败时显示"网络异常，请重试" + 重试按钮

#### 3. 好感度系统
- 初始 50（0-100 范围）
- 正常聊天 +1
- 夸她/关心她 +3~5
- 说冷话/忽略 -2~3
- **判断方式**：LLM 回复末尾返回 `[affection:+1]` 标记，前端解析
- **兜底机制**：如果回复中没解析到标记，默认 +1（防止 LLM 忘记返回）
- **好感度影响：**
  - 50 以下：冷淡
  - 50-70：友好
  - 70-90：亲密
  - 90+：会撒娇

#### 4. AI 对话
- 通过 Vercel Edge Function 代理调用通义/DeepSeek API
- System Prompt 包含角色性格 + 当前好感度
- **对话历史按角色 ID 隔离**（切换角色不会 OOC）
- **IndexedDB 存储**（不是 localStorage，无 5MB 限制）
- 每个角色保留最近 20 条对话

#### 5. 内容安全过滤
- 在 Edge Function 层加 output filter
- 检测敏感词/NSFW 内容，替换为安全回复
- 简单实现：匹配关键词列表，命中则返回"这个话题不太合适呢～"

### P1 - 今天争取做的

#### 6. 语音输入
- **方案 A（推荐）**：引导用户使用输入法自带的语音键盘（零开发，全平台可用）
- **方案 B**：Whisper API 转写（更准确，需 API Key）
- Web Speech API 作为备选（仅 Chrome/Android，支持时显示按钮，不支持时隐藏）
- **用户体验**：输入框旁显示🎤图标，点击后提示"长按麦克风使用输入法语音输入"，首次使用弹引导

#### 7. 语音回复
- **默认模式**：手动点击消息旁的喇叭图标播放（节省 ElevenLabs 额度）
- ElevenLabs TTS（高质量，10000 字符/月免费）
- **备选方案**：浏览器 SpeechSynthesis API（免费但声音机械）

#### 8. 奖励机制
- **双击消息**弹出奖励选项（"摸头"/"抱抱"/"夸奖"）
- 奖励后好感度 +3，角色有专属反应
- 改为双击是因为长按在 iOS/Android 行为不一致
- **用户体验**：首次进入聊天时弹引导"双击消息可以夸她哦～"，或在消息旁加小❤️图标

### P2 - 后续版本

#### 9. 主动发消息
- Demo 阶段：打开页面时模拟收到消息（"早安"/"在干嘛"）
- 完整版：需要后端定时推送，纯前端做不到
- **注意**：Service Worker Push API 必须有服务器支持

#### 10. 全部 16 人格
- 扩展剩余 12 个角色
- 用户可以自定义性格

#### 11. 角色立绘
- 用 ComfyUI 生成每个角色的立绘
- 好感度不同，表情不同

#### 12. 记忆系统
- 记住用户的名字、喜好、说过的话
- 对话中会提到之前聊过的内容

#### 13. 云端同步
- 当前方案：换设备/清缓存 = 数据清零
- Demo 阶段在首次使用时提示用户
- 完整版：用户登录 + 云端存储

## 六、核心数据结构

### 角色数据

```typescript
interface Character {
  id: string;           // "xiaonuan"
  name: string;         // "小暖"
  mbti: string;         // "ENFJ"
  avatar: string;       // "/characters/xiaonuan.png"
  tagline: string;      // "温柔学姐"
  greeting: string;     // "今天过得怎么样呀？"
  personality: string;  // 完整的性格描述（用于 system prompt）
  speechStyle: string;  // 说话风格
}
```

### 好感度

```typescript
interface Affection {
  characterId: string;
  level: number;        // 0-100
  interactions: number; // 总交互次数
  lastChat: string;     // ISO 时间
}
```

### 对话记录（按角色隔离）

```typescript
interface ChatHistory {
  characterId: string;  // 每个角色独立存储
  messages: Message[];  // 最近 20 条
}

interface Message {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  timestamp: string;
  isVoice?: boolean;    // 是否是语音消息
}
```

## 七、System Prompt 模板

```
你是一个叫{name}的女孩，MBTI 人格是{mbti}。

## 你的性格
{personality}

## 你的说话风格
{speechStyle}

## 你和用户的关系
好感度：{affection}/100
你们已经聊了{interactions}次。

好感度低于 50 时，你会保持距离感。
好感度 50-70 时，你会表现得友好但不太主动。
好感度 70-90 时，你会开始关心用户的日常。
好感度 90 以上时，你会撒娇，表达依赖。

## 要求
- 用中文回复
- 每次回复 1-3 句话，不要太长
- 要有感情，像真人一样
- 可以适当用 emoji
- 不要说自己是 AI
- 记住之前对话的内容

## 重要：回复格式
在回复的最后，用方括号标记好感度变化，例如：
[affection:+1] 或 [affection:+3] 或 [affection:-2]
如果不需要变化，返回 [affection:0]
这个标记不会显示给用户，只有系统会读取。
```
