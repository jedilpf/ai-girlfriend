# AI Girlfriend - Demo 策划案 v1

> **目标：今天能跑起来的 Demo**
> **技术路线：PWA（手机浏览器可用）+ API 对话 + 语音输入/输出 + localStorage**
> 
> **v1.1 更新（2026-04-17 12:33）：根据团队反馈优化了三个问题**
> - 语音输入兼容性（iOS Safari 不支持 Web Speech API → 改用输入法麦克风/Whisper）
> - 好感度判断逻辑（关键词匹配 → LLM 返回标记，零额外成本）
> - ElevenLabs TTS 额度限制（自动播放 → 手动点喇叭播放）

---

## 一、产品定位

**一句话：** 一个能和你聊天、有性格、会记住你的 AI 女友。

**核心卖点：**
- MBTI 16 种人格可选，每种性格完全不同
- 能语音说话，不只是打字
- 有好感度，越聊越亲近
- 可以夸她、奖励她，她会有反应
- 她会主动找你说话

**不做的事（Demo 阶段）：**
- ❌ 原生 APP（以后套壳就行）
- ❌ 本地跑模型（API 更快）
- ❌ 多人/社交（单人体验）
- ❌ 付费/商业化

---

## 二、技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 框架 | React + TypeScript + Vite | 你最熟的栈 |
| PWA | vite-plugin-pwa | 一键生成，手机能装 |
| UI | Tailwind CSS v4 | 和 ulive 一样的技术 |
| AI 对话 | 通义 DashScope / DeepSeek API | 便宜、快、中文好 |
| 语音输入 | 输入法麦克风 / Whisper API | iOS 兼容，零开发 |
| 语音输出 | ElevenLabs TTS / SpeechSynthesis API | 手动播放省额度 |
| 数据存储 | localStorage + IndexedDB | 本地，不需要后端 |
| 角色图片 | 占位图 → ComfyUI 生成 | 先用 emoji/色块，后续替换 |

## 三、功能清单（按优先级）

### P0 - 必须有的（今天做完就能用）

#### 1. 角色选择页
- 展示 4 个预设角色（先做 4 个，不全 16 个）
- 每个角色显示：头像（占位）、名字、MBTI 类型、一句话介绍
- 点击选择后进入聊天

**建议先做的 4 个：**
| 名字 | MBTI | 性格标签 | 一句话 |
|------|------|---------|--------|
| 小暖 | ENFJ | 温柔学姐 | "今天过得怎么样呀？" |
| 傲娇 | INTP | 毒舌理工女 | "才不是特意等你的。" |
| 元气 | ESFP | 快乐小狗 | "啊啊啊你来啦！！" |
| 御姐 | INTJ | 高冷总裁 | "说吧，什么事。" |

#### 2. 聊天界面
- 消息气泡（左=她，右=你）
- 输入框 + 发送按钮
- 语音输入按钮（🎤）
- 顶部显示角色名 + 好感度
- 打字效果（逐字显示）

#### 3. 好感度系统
- 初始 50（0-100 范围）
- 正常聊天 +1
- 夸她/关心她 +3~5
- 说冷话/忽略 -2~3
- 好感度影响：
  - 50 以下：冷淡
  - 50-70：友好
  - 70-90：亲密
  - 90+：会撒娇

#### 4. AI 对话
- 调用通义/DeepSeek API
- System Prompt 包含角色性格 + 当前好感度
- 对话历史保留（最近 20 条）
- **好感度变化标记**：LLM 回复末尾附加 `[affection:+1]` 等标记，前端解析后更新好感度

### P1 - 今天争取做的

#### 5. 语音输入
- **方案 A（推荐）**：引导用户使用输入法自带的语音键盘（零开发，全平台可用）
- **方案 B**：Whisper API 转写（更准确，需 API Key）
- Web Speech API 作为备选（仅 Chrome/Android 可用）

#### 6. 语音回复
- **默认模式**：手动点击消息旁的喇叭图标播放（节省 ElevenLabs 额度）
- ElevenLabs TTS（高质量，10000 字符/月免费）
- **备选方案**：浏览器 SpeechSynthesis API（免费但声音机械）

#### 7. 奖励机制
- 长按消息可以"摸头"/"抱抱"/"夸奖"
- 奖励后好感度 +3，角色有专属反应

### P2 - 后续版本

#### 8. 主动发消息
- 模拟推送：打开页面时收到"早安"/"在干嘛"
- 完整版：后端定时推送

#### 9. 全部 16 人格
- 扩展剩余 12 个角色
- 用户可以自定义性格

#### 10. 角色立绘
- 用 ComfyUI 生成每个角色的立绘
- 好感度不同，表情不同

#### 11. 记忆系统
- 记住用户的名字、喜好、说过的话
- 对话中会提到之前聊过的内容

## 五、核心数据结构

### 角色数据
```typescript
interface Character {
  id: string;           // "xiaonuan"
  name: string;         // "小暖"
  mbti: string;         // "ENFJ"
  avatar: string;       // "/characters/xiaonuan.png"
  tagline: string;      // "温柔学姐"
  greeting: string;     // "今天过得怎么样呀？"
  personality: string;  // 完整的性格描述（用于 system prompt）
  speechStyle: string;  // 说话风格
}
```

### 好感度
```typescript
interface Affection {
  characterId: string;
  level: number;        // 0-100
  interactions: number; // 总交互次数
  lastChat: string;     // ISO 时间
}
```

### 对话记录
```typescript
interface Message {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  timestamp: string;
  isVoice?: boolean;    // 是否是语音消息
}
```

## 六、System Prompt 模板

```
你是一个叫{name}的女孩，MBTI 人格是{mbti}。

## 你的性格
{personality}

## 你的说话风格
{speechStyle}

## 你和用户的关系
好感度：{affection}/100
你们已经聊了{interactions}次。

好感度低于 50 时，你会保持距离感。
好感度 50-70 时，你会表现得友好但不太主动。
好感度 70-90 时，你会开始关心用户的日常。
好感度 90 以上时，你会撒娇，表达依赖。

## 要求
- 用中文回复
- 每次回复 1-3 句话，不要太长
- 要有感情，像真人一样
- 可以适当用 emoji
- 不要说自己是 AI
- 记住之前对话的内容
```

## 七、API 配置

### 通义 DashScope（推荐，便宜）
- 模型：qwen-plus
- 价格：按 token 计费，很便宜
- 你需要：DASHSCOPE_API_KEY

### DeepSeek（备选）
- 模型：deepseek-chat
- 价格：更便宜
- 你需要：DEEPSEEK_API_KEY

## 八、开发步骤

### Step 1：初始化项目（10 分钟）
```bash
npm create vite@latest ai-girlfriend -- --template react-ts
cd ai-girlfriend
npm install
npm install -D tailwindcss @tailwindcss/vite
npx tailwindcss init -p
```

### Step 2：创建角色数据（15 分钟）
- 写 4 个角色的数据
- 写对应的 system prompt

### Step 3：角色选择页（30 分钟）
- 网格布局展示角色卡片
- 点击后保存选择，跳转到聊天页

### Step 4：聊天界面（45 分钟）
- 消息列表 + 输入框
- 发送消息 → 调 API → 显示回复
- 实现打字效果

### Step 5：好感度系统（20 分钟）
- localStorage 存储
- **好感度变化通过 LLM 回复末尾标记判断**（system prompt 要求返回 `[affection:+1]` 等标记）
- 不额外调 API，一次请求搞定对话+好感度判断

### Step 6：语音输入（15 分钟）
- 优先方案：引导用户使用输入法语音键盘（零开发）
- 备选方案：Whisper API 集成

### Step 7：语音回复（30 分钟）
- ElevenLabs TTS 调用
- 手动播放模式（点喇叭图标）
- 备选 SpeechSynthesis API

### Step 8：奖励机制（20 分钟）
- 长按消息弹出奖励选项
- 触发好感度变化 + 角色反应

### Step 9：PWA 配置（10 分钟）
- vite-plugin-pwa 配置
- 生成 manifest + service worker
